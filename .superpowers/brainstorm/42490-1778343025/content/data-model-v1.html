Data Model

Core entities for brand discovery + campaign management MVP

Entity Relationships
👤 users id, email, password_hash, role, status
🏢 brand_profiles belongs to user
company_name, industry, website, description, logo_url
⭐ influencer_profiles belongs to user
display_name, handle, bio, niche, follower_count, engagement_rate, platforms[], location, profile_image_url
📋 campaigns belongs to brand (user)
title, description, status (draft/active/completed/cancelled), budget, start_date, end_date
📩 invitations belongs to campaign + influencer
status (pending/accepted/declined/withdrawn), message, created_at, responded_at
✅ deliverables belongs to campaign + influencer
description, due_date, status (pending/in_progress/completed), completed_at
🔔 notifications belongs to user
type, title, message, read (bool), related_entity_type, related_entity_id

Key Decisions

  • Separate profile tables — keeps brand and influencer data clean. Adding agencies later means a third profile table, no migration needed.
  • Platforms as JSON — influencers can have multiple social accounts (Instagram, TikTok, YouTube). JSON array is flexible for MVP.
  • Invitation as its own table — not just a campaign status. This supports future features like invitation history, negotiation, bulk invites.
