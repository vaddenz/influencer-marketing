Data Flow: Invitation & Campaign Lifecycle

How state changes flow through the system from discovery to completion

State Machine: Invitation
pending
Brand sends invite
→
accepted
Influencer accepts
→
in_progress
Deliverables start
→
completed
All deliverables done
⤴ pending can also go to declined (influencer declines)
⤴ pending can go to withdrawn (brand cancels)
Sequence: Brand Invites → Influencer Accepts
Brand
POST /campaigns → creates campaign (optional, if new)
Brand
POST /invitations → sends invite (campaign_id, influencer_id, message)
System
Creates notification for influencer (type: invitation_received)
Influencer
GET /invitations → sees pending invitation
Influencer
PATCH /invitations/:id/accept → status becomes accepted
System
Creates notification for brand (type: invitation_accepted)
System
Creates deliverables from campaign template (status: pending)
Influencer
PATCH /deliverables/:id/complete → marks deliverable done
System
When all deliverables complete → notification to brand (type: deliverables_completed)
